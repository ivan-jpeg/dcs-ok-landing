# Задача: утилита измерения максимальной перегрузки при посадке в DCS

Реализовать утилиту для симулятора Digital Combat Simulator (DCS), которая измеряет максимальную вертикальную перегрузку при посадке (касании земли) самолёта игрока.

---

## Окружение

Утилита должна работать как **исполняемый Lua-скрипт** в окружении DCS (не мод и не плагин — именно скрипт, выполняемый в среде симулятора). Использовать DCS Scripting API.

---

## Бизнес-логика

### Начало измерения

В **радио-меню** в разделе **Other** есть пункт меню **Старт измерения**, при выборе этого пункта выводится **стандартное окно сообщения DCS**, в котором отображается **максимальная достигнутая с момента инициализации вертикальная перегрузка**.

Текстовое содержимое окна (точно в таком виде):

```
Максимальная перегрузка
______________________

Ny = 00.00
```

- Значение Ny — максимум вертикальной перегрузки с момента инициализации или последнего сброса.
- Формат числа: два знака после запятой (например, 00.00, 02.35).

### Сброс измерения

В **радио-меню** в разделе **Other** есть пункт меню **«Сброс измерения»**. При выборе этого пункта:
- достигнутое максимальное значение сбрасывается;
- замер продолжается заново (накопление нового максимума).

### Конец измерения

В **радио-меню** в разделе **Other** есть пункт меню **«Стоп измерения»**. При выборе этого пункта:
- измерение прекращается;
- окно сообщения скрывается.

---

## Результат

Сгенерированный Lua-скрипт сохранить в файл **`ok-landing.lua`** в директории **`ok-landing`** (относительно корня проекта). Итоговый путь: `ok-landing/ok-landing.lua`.

---

## Технические заметки для исполнителя

- Код скрипта должен быть пригоден для вызова в мисси по команде DO SCRIPT FILE
- Использовать DCS Scripting API для:
  - радио-меню (добавление пунктов в раздел /Other);
  - стандартных окон сообщений (показ и скрытие);
  - данных о перегрузке (вертикальная перегрузка Ny).
- Источник данных о перегрузке — актуальный API DCS (например, `LoGetMCPState()` или эквивалент в текущей версии DCS scripting; при необходимости уточнить по документации).
- Учитывать только единицу (самолёт) игрока; привязка логики к самолёту игрока обязательна.
